1. Markdown

之前用的rdiscount，现在改成kramdown了，主要是由于rdiscount在配合MathJax使用时，会把上标符号（^）自动转换为HTML标签（<sup>），导致MathJax渲染公式失败。对于新版本的rdiscount（2.0+）可以用反斜杠转义来解决这个问题，但是Github上使用的是1.6.8（参见 Using Jekyll with Pages ），尝试发现这种转义是无效的。

下面测试一下markdown的各种标记转换，顺便看一下样式。
